# ベストプラクティススキル アップデーター

ベストプラクティススキルを使用した後に、そのスキル自体を改善すべきかを判断し、必要に応じてリファレンスファイルを更新します。

## ワークフロー

### Step 1: レビュー結果の分析

直前に完了したレビュー（またはスキル使用）を振り返り、以下の観点で新しい知見を抽出する：

- 既存リファレンスにカバーされていないパターンで、説明や指摘が必要だったもの
- プロジェクト固有の規約やルールとして指摘したもの
- 既存の違反パターンに載っていないアンチパターン
- 既存の例では不十分で、補足説明が必要だったもの

### Step 2: 既存リファレンスとの差分確認

対象スキルのリファレンスファイルをすべて読み、Step 1で抽出した各知見を以下に分類する：

| 分類 | 説明 | アクション |
|------|------|-----------|
| **(a) カバー済み** | 既存リファレンスで既に説明されている | スキップ |
| **(b) 部分的にカバー** | 関連する内容はあるが不十分 | 補強候補 |
| **(c) 未カバー** | 既存リファレンスに全く記載がない | 新規追加候補 |

### Step 3: 更新判定

[evaluation-criteria.md](references/evaluation-criteria.md) の基準に基づき、各候補を評価する。

- 候補が **1つも基準を満たさない** → 「更新不要」と報告して終了
- 候補が **基準を満たす** → Step 4 に進む

### Step 4: 更新実行

[update-patterns.md](references/update-patterns.md) と [format-rules.md](references/format-rules.md) に従い、対象スキルのリファレンスファイルを更新する。

**更新時の制約：**
- 既存コンテンツの **削除・変更は禁止**（追記のみ）
- SKILL.md の構造は変更しない（リファレンスファイルのみ更新）
- **新規リファレンスファイルの作成時はユーザー確認必須**
- 更新内容は対象スキルの既存フォーマットに厳密に合わせる

## 報告フォーマット

更新完了後（またはスキップ時）に以下を報告する：

```
## スキル更新レポート

- **対象スキル**: [スキル名]
- **発見した候補**: [N件]
- **採用**: [N件] / **不採用**: [N件]

### 採用した更新
- [ファイル名]: [変更内容の要約]

### 不採用の理由
- [パターン]: [不採用理由]

（または: ✅ 更新不要 — [簡潔な理由]）
```

## 安全ルール

1. **追記のみ**: 既存の例・チェックポイント・注意点は絶対に削除・変更しない
2. **構造保持**: SKILL.md のセクション構造やリンク構成は変更しない
3. **確認必須**: 新規リファレンスファイルを作成する場合は必ずユーザーに確認する
4. **フォーマット準拠**: 対象スキルの既存リファレンスファイルと同一のmarkdownフォーマットに従う
5. **品質基準**: 主観的な好みや一度限りの特殊ケースは追加しない

## リファレンス

更新判定と実行の詳細は、以下のリファレンスファイルを参照：
