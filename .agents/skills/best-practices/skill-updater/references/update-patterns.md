# 許可される更新パターン

**原則**: スキルの品質を維持しながら、リファレンスファイルに新しい知見を安全に追加する

## パターン一覧

| # | パターン | 対象セクション | ユーザー確認 |
|---|---------|-------------|------------|
| 1 | 違反パターン追加 | `## よくある違反パターン` | 不要 |
| 2 | チェック観点追加 | `## チェック観点` | 不要 |
| 3 | チェックポイント追加 | `## コードレビュー時のチェックポイント` | 不要 |
| 4 | 注意点追加 | `## 注意点` | 不要 |
| 5 | 新規リファレンスファイル作成 | 新ファイル + SKILL.md | **必要** |
| 6 | 既存例の補足 | 既存の違反例/改善例の直後 | 不要 |

---

## パターン 1: 違反パターン追加

既存リファレンスファイルの `## よくある違反パターン` セクション末尾に追加する。

**追記テンプレート:**

```markdown
### パターンN: [パターン名]

[1〜2文の説明]

❌ 悪い例:
\```[言語]
# 悪い例のコード
\```

✅ 良い例:
\```[言語]
# 良い例のコード
\```
```

**ルール:**
- パターン番号は既存の最後の番号 + 1 とする
- 悪い例と良い例は必ずペアで記載する
- コードブロックの言語タグは対象スキルに合わせる

## パターン 2: チェック観点追加

既存リファレンスファイルの `## チェック観点` セクション末尾に追加する。

**追記テンプレート:**

```markdown
- [新しいチェック観点の説明]
```

**ルール:**
- 既存のチェック観点と重複しないこと
- 1行で簡潔に表現する
- 既存の粒度・トーンに合わせる

## パターン 3: チェックポイント追加

既存リファレンスファイルの `## コードレビュー時のチェックポイント` セクション末尾に追加する。

**追記テンプレート:**

```markdown
N. **[キーワード]**: [チェックポイントの説明]
```

**ルール:**
- 番号は既存の最後の番号 + 1 とする
- キーワードは太字にする
- 具体的で検証可能な内容にする

## パターン 4: 注意点追加

既存リファレンスファイルの `## 注意点` セクション末尾に追加する。

**追記テンプレート:**

```markdown
- **[キーワード]**: [注意点の説明]
```

**ルール:**
- キーワードは太字にする
- 既存の注意点と矛盾しないこと

## パターン 5: 新規リファレンスファイル作成

レビューで既存ファイルのどのカテゴリにも属さない、まとまったベストプラクティス群が発見された場合に使用する。

**⚠️ ユーザー確認が必須**

**手順:**

1. ユーザーに新規ファイルの作成を提案し、ファイル名とカバー範囲を説明する
2. ユーザーの承認を得る
3. 以下のテンプレートでファイルを作成する
4. 親スキルの SKILL.md に新ファイルへのリンクを追加する

**新規リファレンスファイルテンプレート:**

```markdown
# [トピック]のベストプラクティス

**原則**: [1文で原則を記述]

[1〜2文の導入説明]

## チェック観点

- [チェック項目1]
- [チェック項目2]
- [チェック項目3]

## 違反例

### 例1: [違反の名前]

\```[言語]
# ❌ 悪い例: [説明]
[コード]
\```

## 改善例

### 例1: [改善の名前]

\```[言語]
# ✅ 良い例: [説明]
[コード]
\```

## よくある違反パターン

### パターン1: [パターン名]

[説明]

❌ 悪い例:
\```[言語]
[コード]
\```

✅ 良い例:
\```[言語]
[コード]
\```

## コードレビュー時のチェックポイント

1. **[キーワード]**: [説明]
2. **[キーワード]**: [説明]

## 注意点

- **[キーワード]**: [説明]
```

**SKILL.md への追記:**

レビュープロセスの参照リスト、チェックリスト、リファレンスリストの各セクションに新ファイルへのリンクを追加する。

## パターン 6: 既存例の補足

既存の違反例や改善例が正しいが不十分な場合、追加の例やコメントを直後に補足する。

**追記テンプレート:**

```markdown
### 例N-b: [補足の名前]

\```[言語]
# [❌/✅] [説明]
[補足コード]
\```
```

**ルール:**
- 既存の例を変更・削除しない
- 補足であることが分かる番号付けにする（例: 例1-b）
- 既存の例と矛盾しないこと
