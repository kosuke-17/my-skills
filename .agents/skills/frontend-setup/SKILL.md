# フロントエンド環境構築スキル

決めうちの技術スタックでプロジェクトの開発環境を構築する。

## 利用可能なスタック

| スタックファイル                         | フレームワーク       | 用途                                       |
| ---------------------------------------- | -------------------- | ------------------------------------------ |
| `references/nextjs-stack.md`             | Next.js (App Router) | SSR / フルスタック / SEO が必要なアプリ    |
| `references/react-vitest-stack.md`       | Vite + React         | SPA / ライブラリ / シンプルなフロントエンド |

> **スタックを追加・変更したい場合は `references/` に新しいスタックファイルを追加し、このテーブルを更新する。**

---

## セットアップフロー

**必ず Phase 0 から順に実行すること。**

### Phase 0: スタック選択

0-1. 利用可能なスタック一覧をユーザーに提示する
0-2. ユーザーが使用するスタックを選択する
0-3. 選択されたスタックファイルを読み込む（以降の手順はそのファイルの定義に従う）

### Phase 1: 事前確認

1-1. 対象ディレクトリが空であること（または新規作成先）を確認する
1-2. Node.js がインストールされていることを確認する（`node -v`）
1-3. プロジェクト名をユーザーに確認する

### Phase 2: プロジェクト作成

2-1. pnpm がインストールされていることを確認する（スタックファイルの `パッケージマネージャー.check` を実行）。未インストールならユーザーに案内して停止する
2-2. フレームワークのプロジェクトを作成する（スタックファイルの `フレームワーク.command` を実行）
2-3. TypeScript の `strict: true` を `tsconfig.json` で確認する

### Phase 3: フォーマッター・リンター設定

3-1. ESLint をインストールする（スタックファイルの `リンター.command` を実行）
3-2. Prettier と関連パッケージをインストールする（スタックファイルの `フォーマッター.command` を実行）
3-3. 各設定ファイルを作成する（スタックファイルの `.config` に従う）

### Phase 4: CSS

4-1. Tailwind CSS をインストール・設定する（スタックファイルの `CSS.command` を実行）
4-2. スタックファイルの `CSS.note` に従い追加設定を行う

### Phase 5: テスト環境

5-1. Vitest と関連パッケージをインストールする（スタックファイルの `テスト.command` を実行）
5-2. テスト設定ファイルを作成・更新する（スタックファイルの `テスト.config` に従う）
5-3. `package.json` にテスト用スクリプトを追加する（スタックファイルの `テスト.scripts` に従う）

### Phase 6: Storybook & Chromatic

6-1. Storybook を初期化する（スタックファイルの `Storybook.command` を実行）
6-2. Chromatic をインストールする（スタックファイルの `Chromatic.command` を実行）
6-3. `package.json` に chromatic スクリプトを追加する（スタックファイルの `Chromatic.scripts` に従う）

### Phase 7: 最終確認

7-1. `pnpm install` で依存関係が正常にインストールされることを確認
7-2. `pnpm build` でビルドが通ることを確認
7-3. `pnpm test:ci` でテストが実行可能なことを確認
7-4. セットアップ結果のサマリーをユーザーに提示する

---

## フロー制御ルール

1. **必ず Phase 0 から順に実行する。** フェーズをスキップしない。
2. **各コマンドの実行結果を確認してから次へ進む。** エラーが出た場合はその場で対処する。
3. **部分実行にも対応する。** ユーザーが「テスト環境だけ追加して」と言った場合は、該当フェーズだけを実行する。
4. **コマンドは選択したスタックファイルの定義に従う。** スタックファイルに書かれていないコマンドを勝手に追加しない。
