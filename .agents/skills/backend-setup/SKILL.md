# バックエンド環境構築スキル

決めうちの技術スタックでプロジェクトの開発環境を構築する。

## 利用可能なスタック

| スタックファイル                         | フレームワーク | 用途                                           |
| ---------------------------------------- | -------------- | ---------------------------------------------- |
| `references/fastapi-stack.md`            | FastAPI        | REST API / 非同期処理 / 高パフォーマンスな API |
| `references/nestjs-stack.md`             | NestJS         | TypeScript / DI / エンタープライズ向け API     |
| `references/gin-stack.md`               | Gin            | Go / シンプルな高パフォーマンス REST API        |

> **スタックを追加・変更したい場合は `references/` に新しいスタックファイルを追加し、このテーブルを更新する。**

---

## セットアップフロー

**必ず Phase 0 から順に実行すること。**

### Phase 0: スタック選択

0-1. 利用可能なスタック一覧をユーザーに提示する
0-2. ユーザーが使用するスタックを選択する
0-3. 選択されたスタックファイルを読み込む（以降の手順はそのファイルの定義に従う）

### Phase 1: 事前確認

1-1. 対象ディレクトリが空であること（または新規作成先）を確認する
1-2. Python がインストールされていることを確認する（`python3 --version`）
1-3. プロジェクト名をユーザーに確認する

### Phase 2: プロジェクト作成

2-1. パッケージマネージャーがインストールされていることを確認する（スタックファイルの `パッケージマネージャー.check` を実行）。未インストールならユーザーに案内して停止する
2-2. プロジェクトを初期化する（スタックファイルの `パッケージマネージャー.init_command` を実行）
2-3. フレームワークと主要な依存パッケージをインストールする（スタックファイルの `フレームワーク.command` を実行）
2-4. プロジェクトの基本ディレクトリ構造を作成する（スタックファイルの `フレームワーク.structure` に従う）
2-5. エントリーポイントファイルを作成する（スタックファイルの `フレームワーク.entrypoint` に従う）

### Phase 3: フォーマッター・リンター設定

3-1. リンター・フォーマッターをインストールする（スタックファイルの `リンター・フォーマッター.command` を実行）
3-2. 設定ファイルを作成する（スタックファイルの `リンター・フォーマッター.config` に従う）

### Phase 4: データベース

4-1. データベース関連パッケージをインストールする（スタックファイルの `データベース.command` を実行）
4-2. データベース設定ファイルを作成する（スタックファイルの `データベース.config` に従う）
4-3. マイグレーションツールを設定する（スタックファイルの `データベース.migration` に従う）

### Phase 5: テスト環境

5-1. テスト関連パッケージをインストールする（スタックファイルの `テスト.command` を実行）
5-2. テスト設定を追加する（スタックファイルの `テスト.config` に従う）
5-3. テスト用ディレクトリとサンプルテストを作成する（スタックファイルの `テスト.structure` に従う）

### Phase 6: Docker & Docker Compose

6-1. Dockerfile を作成する（スタックファイルの `Docker.dockerfile` に従う）
6-2. docker-compose.yml を作成する（スタックファイルの `Docker.compose` に従う）
6-3. .dockerignore を作成する（スタックファイルの `Docker.dockerignore` に従う）

### Phase 7: 最終確認

7-1. 依存関係が正常にインストールされることを確認（スタックファイルのパッケージマネージャーに従う）
7-2. リンター・フォーマッターが正常に動作することを確認
7-3. `docker compose build` でビルドが通ることを確認
7-4. セットアップ結果のサマリーをユーザーに提示する

---

## フロー制御ルール

1. **必ず Phase 0 から順に実行する。** フェーズをスキップしない。
2. **各コマンドの実行結果を確認してから次へ進む。** エラーが出た場合はその場で対処する。
3. **部分実行にも対応する。** ユーザーが「テスト環境だけ追加して」と言った場合は、該当フェーズだけを実行する。
4. **コマンドは選択したスタックファイルの定義に従う。** スタックファイルに書かれていないコマンドを勝手に追加しない。
