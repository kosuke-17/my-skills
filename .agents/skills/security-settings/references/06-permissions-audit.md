# 権限の棚卸し - /permissionsによる定期確認

## なぜ必要か

Claude Codeを長く使っていると、セッション中に「Always allow」で許可したルールが蓄積する。
放置すると意図しない権限が残り、セキュリティ上のリスクになる。
**月1回程度の棚卸し**で不要なallowルールを削除し、設定の健全性を保つ。

---

## チェック方法

棚卸しの実施状況は設定ファイルの内容から間接的に判断する。

確認ポイント：

1. `permissions.allow` に不審なルールが蓄積していないか
2. `deny` ルールが意図通りに設定されているか
3. 設定ファイルの更新日時（最近棚卸しされているか）

### 判定基準

| 状態 | 判定 |
|------|------|
| `permissions.allow` が空または最小限、`deny` が適切に設定されている | 🟢 設定済み |
| `permissions.allow` に不審なルールがある、または `deny` が少なすぎる | 🟡 部分的 |
| `permissions.allow` が大量に蓄積し、広範な権限が付与されている | 🔴 危険 |

---

## 棚卸しの手順

### 1. /permissionsで現在の設定を確認

Claude Codeのチャット内で実行：

```
/permissions
```

現在有効な全パーミッション設定が表示される。

### 2. /statusで読み込まれている設定ファイルを確認

```
/status
```

どの設定ファイルが読み込まれているか、エラーがないかを確認できる。

### 3. 不要なallowルールを削除

`settings.json` または `settings.local.json` から不要なルールを削除する。

---

## allowルールが危険なパターン

以下のような広範なallowルールが残っている場合は削除を検討：

```json
{
  "permissions": {
    "allow": [
      "Bash(*)",          // ⚠️ 全Bashコマンドを許可
      "Bash(git *)",      // ⚠️ 全gitコマンドを許可
      "Write(**/*)",      // ⚠️ 全ファイルへの書き込みを許可
      "Read(**/*)"        // ⚠️ 全ファイルの読み取りを許可
    ]
  }
}
```

---

## ConfigChangeフックの活用（最新機能）

最新バージョンでは `ConfigChange` フックが利用可能。
権限設定が変更されたタイミングで通知や監査ログ記録ができる。

```json
{
  "hooks": {
    "ConfigChange": [
      {
        "matcher": "permissions",
        "hooks": [{
          "type": "command",
          "command": "echo '権限設定が変更されました' >> ~/.claude/audit.log"
        }]
      }
    ]
  }
}
```

チーム開発では特に有効。

---

## 推奨サイクル

- **個人利用**: 月1回、`/permissions` コマンドで確認
- **チーム利用**: `ConfigChange` フックで変更を自動記録 + 月次レビュー
- **センシティブなプロジェクト**: 作業開始時と終了時に確認
