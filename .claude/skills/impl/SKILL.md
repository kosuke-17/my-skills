# CDD Implementation

Implement components bottom-up based on a plan file.

## Loading the Plan File

- With argument: Read the path from `$ARGUMENTS`
- Without argument: Use the latest plan file

Latest plan file: !`ls -t .claude/plans/*.plan.md 2>/dev/null | head -1 || echo "No plan files found in .claude/plans/. Run /plan first."`

If no plan file is found, instruct the user to run `/plan` first.

After loading the plan file, register each component from the "Implementation Order" section into TodoWrite.

## Implementation Cycle per Component

Follow the plan's implementation order and implement each component through the following phases.

### Phase 1: Static UI (index.tsx)

Complete the visual appearance 100% with props only. **Do not use state.**

```tsx
import { cn } from '@/utils/cn'

type ComponentNameProps = {
  label: string
  variant?: 'primary' | 'secondary'
  className?: string
}

export default function ComponentName({
  label,
  variant = 'primary',
  className,
}: ComponentNameProps) {
  return (
    <div className={cn('base-styles', className)}>
      {label}
    </div>
  )
}
```

**Checks:**

- No `useState` used
- UI is complete with props alone
- Classes composed with `cn()`
- CVA variants configured if needed

### Phase 2: Story Creation (index.stories.tsx)

**Before writing any Story**, call `mcp__storybook-mcp__get-storybook-story-instructions` to get the latest Storybook conventions for this project.

If the component reuses existing components, call `mcp__storybook-mcp__get-documentation` for those components to understand their props and Story patterns.

Create Stories in CSF 3 format.

```tsx
import type { Meta, StoryObj } from '@storybook/react'

import ComponentName from './index'

const meta: Meta<typeof ComponentName> = {
  title: 'Components/ComponentName',
  component: ComponentName,
  tags: ['autodocs'],
}
export default meta

type Story = StoryObj<typeof ComponentName>

export const Default: Story = {
  args: {
    label: 'Label',
  },
}
```

**After writing the Story**, verify it works:

1. Call `mcp__storybook-mcp__preview-stories` to get the preview URL and confirm rendering
2. Call `mcp__storybook-mcp__run-story-tests` with the specific story to run tests (including a11y)

**Checks:**

- `tags: ['autodocs']` included
- Stories cover all planned variations
- Fixed values used (no random values)
- Story renders correctly (verified via preview)
- Story tests pass (verified via run-story-tests)

### Phase 3: Unit Test (index.test.tsx)

Create tests with Arrange-Act-Assert structure.

```tsx
import { render, screen } from '@testing-library/react'

import ComponentName from './index'

describe('ComponentName', () => {
  it('renders correctly', () => {
    // Arrange
    const props = { label: 'Test' }

    // Act
    render(<ComponentName {...props} />)

    // Assert
    expect(screen.getByText('Test')).toBeInTheDocument()
  })
})
```

### Phase 4: Add Dynamic UI (only if needed)

Only execute if the plan includes dynamic UI.

1. **Create custom hook** (hooks.ts): Separate state and logic

```tsx
// hooks.ts
export function useComponentLogic() {
  const [isOpen, setIsOpen] = useState(false)
  const toggleOpen = useCallback(() => setIsOpen((prev) => !prev), [])
  return { isOpen, toggleOpen }
}
```

2. **Update component**: Add state using the hook
3. **Add play function tests**: Add interaction tests to Stories

```tsx
import { expect, userEvent, within } from '@storybook/test'

export const WithInteraction: Story = {
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement)
    await userEvent.click(canvas.getByRole('button'))
    await expect(canvas.getByText('Result')).toBeInTheDocument()
  },
}
```

## File Structure Convention

```
components/{component-name}/
├── index.tsx           # Main component
├── index.stories.tsx   # Storybook Story (CSF 3, tags: ['autodocs'])
├── index.test.tsx      # Jest unit test
└── hooks.ts            # Custom hooks (only if needed)
```

Same structure for `features/{feature-name}/components/{component-name}/`.

## Coding Conventions

| Item | Convention |
| -------------- | ----------------------------------------------- |
| File names | kebab-case directories, `index.tsx` entry |
| Imports | Prefer `@/` aliases |
| Styling | Tailwind CSS + `cn()` |
| Variants | CVA (`class-variance-authority`) |
| Exports | `export default` |
| Boolean props | `is`/`has`/`can` prefix |
| Semicolons | None |
| Quotes | Single quotes |
| Trailing commas | ES5 |
